# 第一篇 Markdown 示例文章

这是一篇已经接入到站点里的 Markdown 文章示例。以后你不需要再手写整页 HTML，只要专注写内容本身。

## 现在的发布方式

1. 在 `posts/` 目录里新增一个 `.md` 文件
2. 在 `posts/manifest.json` 里补一条文章信息
3. 执行 `git add .`
4. 执行 `git commit -m "publish new post"`
5. 执行 `git push`

推送完成后，GitHub Pages 会自动更新线上站点。

## 支持什么格式

- 一级到六级标题
- 无序列表和有序列表
- 引用块
- 行内代码，比如 `git push`
- 代码块
- 链接，比如 [GitHub](https://github.com/fffop)

## 一个简单的代码块示例

```powershell
git add .
git commit -m "publish new post"
git push
```

> 建议你把每篇文章写成“做了什么、为什么这么做、下一步做什么”，这样会很像你自己的 log。

## 下一步怎么用

最简单的办法是复制这篇文章，然后改文件名、标题和正文。
